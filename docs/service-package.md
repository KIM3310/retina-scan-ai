# Service Package - retina-scan-ai

This package defines the most practical service shape for this repository. It is public-safe: no price promises, financial projections, or buyer-specific contract assumptions are published here.

## Buyer and Wedge

- **Service lane:** Research validation workflow
- **Primary buyer:** Research groups, health-tech prototype teams, ML reviewers, and model governance teams
- **Front-door offer:** Non-diagnostic validation study
- **Service outcome:** Package medical-image model review as research validation with explicit non-clinical boundaries.

## Build and Polish Priorities

- Keep non-diagnostic language visible
- Clarify approved dataset and reviewer ownership
- Polish validation report structure

## Buyer-Ready Deliverables

- Dataset protocol review
- Research validation
- Model governance report

## Activation Path

1. Use the public demo, README, screenshots, and tests to establish proof.
2. Confirm the buyer workflow, data boundary, owner, and acceptance criteria.
3. Build only the smallest workflow needed to prove the service outcome.
4. Add cloud resources, accounts, storage, or paid integrations only after the workflow boundary is approved.

## Margin and Risk Controls

- Keep research-only scope and avoid unsupported clinical claims.
- Keep the first service scope narrow, repeatable, and evidence-led.
- Keep private financial planning outside the public repository.
- Keep logs, analytics, and reports free of private payloads unless explicit approval exists.

## Portfolio Signal

The portfolio should present this repository as **Non-diagnostic validation study** under the **Research validation workflow** lane, with proof links routed to the live demo or repository evidence.
